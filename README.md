# 📋 Система управління графіком чергувань (Django)

Демонстраційний веб-додаток для управління графіком чергувань працівників на різних об'єктах.

## 🎯 Функціональність

- ✅ Перегляд графіка чергувань у вигляді таблиці
- ✅ Додавання нових чергувань
- ✅ Управління списком працівників
- ✅ Управління списком об'єктів
- ✅ Збереження даних у базі MariaDB
- ✅ Календарний вигляд чергувань
- ✅ Адаптивний дизайн з бічною панеллю

## 📦 Технології

- **Backend**: Django 5.0
- **База даних**: MariaDB + mysqlclient
- **Frontend**: HTML, CSS (адаптивний дизайн)
- **Мова інтерфейсу**: Українська

## 🚀 Швидкий старт

### 1. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 2. Налаштування бази даних MariaDB

1. Встановіть MariaDB з https://mariadb.org/download/
2. Створіть базу даних:
```sql
CREATE DATABASE duty_schedule CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

3. Створіть файл `.env` на основі `.env.example`:
```bash
copy .env.example .env
```

4. Відредагуйте `.env` і вкажіть ваші дані для підключення до MariaDB:
```
DB_NAME=duty_schedule
DB_USER=root
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=3306
```

### 3. Міграції бази даних

```bash
python manage.py makemigrations
python manage.py migrate
```

### 4. Створення суперкористувача (опціонально)

```bash
python manage.py createsuperuser
```

### 5. Запуск додатку

#### Варіант 1: Через bat-файл
Просто двічі клацніть на файл `start.bat`.

#### Варіант 2: Через командний рядок
```bash
python manage.py runserver
```

або

```bash
py manage.py runserver
```

### 6. Відкрийте браузер

Перейдіть за адресою:
```
http://127.0.0.1:8000
```

Адмін-панель доступна за адресою:
```
http://127.0.0.1:8000/admin/
```

## 💾 База даних

Додаток використовує MariaDB з трьома таблицями:

### Employee (Працівники)
- `id` - унікальний ідентифікатор
- `name` - ім'я та прізвище
- `position` - посада
- `created_at` - дата створення

### Site (Об'єкти)
- `id` - унікальний ідентифікатор
- `name` - назва об'єкта
- `address` - адреса
- `created_at` - дата створення

### Duty (Чергування)
- `id` - унікальний ідентифікатор
- `employee_id` - посилання на працівника
- `site_id` - посилання на об'єкт
- `date` - дата чергування
- `notes` - примітки
- `created_at` - дата створення

## 📝 Використання

### Додавання працівника
1. Перейдіть в розділ "Працівники"
2. Натисніть "+ Додати працівника"
3. Заповніть форму (ім'я обов'язкове)
4. Натисніть "Зберегти"

### Додавання об'єкта
1. Перейдіть в розділ "Об'єкти"
2. Натисніть "+ Додати об'єкт"
3. Заповніть форму (назва обов'язкова)
4. Натисніть "Зберегти"

### Додавання чергування
1. Натисніть "Додати чергування" в бічній панелі
2. Оберіть працівника зі списку
3. Оберіть об'єкт зі списку
4. Вкажіть дату початку та дату закінчення
5. За бажанням додайте примітки
6. Натисніть "Зберегти"

### Перегляд календаря
1. Натисніть "Календар" в бічній панелі
2. Переглядайте чергування на місяць
3. Використовуйте стрілки для навігації між місяцями

## 🔧 Налаштування

### Зміна порту
```bash
python manage.py runserver 8080
```

### Production режим
1. Встановіть `DEBUG=False` у `.env`
2. Налаштуйте `ALLOWED_HOSTS` у `settings.py`
3. Використовуйте WSGI сервер (gunicorn, uwsgi)
